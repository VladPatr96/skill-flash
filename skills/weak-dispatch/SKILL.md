# Weak-dispatch: усилитель слабых моделей

Слабая модель проваливает четыре вещи: длинный контекст, решения на развилках, само-оценку, остановку вовремя. Все четыре заменяются процессом — исполнителю остаётся короткий однозначный шаг с внешней проверкой.

## Правила диспатча (поверх обычного конвейера office)

1. **Атомарность.** Задача = один файл, ожидаемый дифф ≤ ~40 строк. Крупнее — дроби на цепочку спек (issues или файлы) ДО диспатча, не надейся.
2. **Контекст-пакет.** Вырезки затрагиваемых функций — прямо в тело спеки (добудь через `graphify query`, а без graphify — вырезками от скаута). Weak-исполнитель не ищет и не читает ничего сверх спеки.
3. **Машинные гейты.** Команды DoD (lint / typecheck / tests) исполнитель прогоняет сам и сдаёт verifier'у только зелёное, вывод гейтов — в отчёт.
4. **Best-of-N.** Рискованная задача или первый провал: 2–3 параллельных исполнителя в ворктри на одну спеку; приёмка первого прошедшего DoD, остальные ветки удалить.
5. **Tracer bullet.** Для неопределённой реализации сначала вертикальный минимальный срез, который проходит один EARS-критерий, а не полный дизайн.
6. **Два реджекта -> auto-debug.** После двух FAIL от verifier не давай третью попытку тому же исполнителю. Сначала diagnostician превращает факты в правку спеки.
7. **Диагност между провалами.** После провала верификации переведи факты verifier'а в правку спеки («падает X, потому что Y → добавь шаг Z») — только потом ретрай. «Не работает, попробуй ещё» — запрещённый промт.
8. **Implementation Notes.** Исполнитель пишет только факты, важные следующей атомарной задаче: изменённый контракт, новая команда, обнаруженная ловушка.
9. **Запрет творчества.** В конверте: не менять подписи/имена/структуру сверх спеки; сомнение = стоп и доклад.

## Конверт (добавить к обычному конверту диспатча)

```
Профиль: weak. Правила поверх спеки:
— меняй ТОЛЬКО файл из спеки; ожидаемый дифф ≤40 строк, выходит больше — СТОП и доклад;
— контекст в спеке полный (вырезки приложены) — ничего не ищи сверх него;
— перед отчётом прогони гейты из DoD: <команды>; verifier'у сдаётся только зелёное,
  вывод гейтов приложи к отчёту;
— чек-лист: (1) дифф только в целевом файле, (2) тест шага 1 красный→зелёный,
  (3) комментарии на русском, (4) graphify update . прогнан (если graphify используется);
— никакого рефакторинга сверх спеки; сомнение = вопрос в отчёте, не импровизация.
— если verifier дважды отклонил задачу, третью попытку не начинай без обновлённой спеки;
— Implementation Notes: только факты для следующей задачи, без советов и планов.
```

## Калибровка

Новая дешёвая модель → бенчмарк: та же реальная задача «голой» моделью vs под профилем. Метрики: % PASS verifier с первого раза, итераций до PASS. Результат — в lessons.md.
