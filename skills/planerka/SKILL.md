# Планёрка

Одно сообщение фаундеру: факты → решения пакетом → что поедет дальше. Время фаундера — бюджет №1: не дёргать по одному вопросу.

## Процедура

1. Прочитай `company/state.md` (конвейер, «Ждут пользователя»), `company/inbox.md` (если есть) и свежие закрытия issues с прошлой планёрки (`gh issue list -s closed`).
2. Собери сообщение по шаблону:
   - **Что произошло** — 3-6 строк фактов: закрытые issues + вердикты, новые blocked, расход лимитов. Без пересказа отчётов агентов.
   - **Решения (≤5)** — каждое: вопрос одной строкой + рекомендация + обоснование + цена (unit-оценка, если найм/инструмент/этап). Задавай через AskUserQuestion, рекомендация — первой опцией. Блокирующие скоуп/деньги/наружу сюда не копятся — они задаются сразу при возникновении.
   - **Поедет дальше** — очередь конвейера после решений.
3. Молчание фаундера по пункту = рекомендация принята — зафиксируй «принято по умолчанию» с обоснованием (принцип прозрачности).
4. Реши больше 5 вопросов накопилось — лишние остаются в «Ждут пользователя» до следующей планёрки, скажи об этом.

## Выход (DoD)

- Протокол в `company/meetings/YYYY-MM-DD-planerka.md` (решения + кто принял: фаундер/по умолчанию).
- `company/state.md` обновлён: решения перенесены из «Ждут пользователя», конвейер актуален.
- `company/inbox.md` очищен (обработанное — в протокол).
- Диспатчи, разблокированные решениями, — запущены.
