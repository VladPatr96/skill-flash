<!-- Шаблон тела issue (спеки). Тело переписывается целиком: `gh issue edit N --body`. -->

**Цель:** одно предложение — что видит пользователь после мержа.

**Контекст:** файлы и строки, которые менять; контракты (форматы сообщений, схемы); ловушки (дубли, генерённые файлы). Всё, что нужно исполнителю, ЧТОБЫ НЕ ИССЛЕДОВАТЬ САМОМУ. Источники контекста: `graphify query` + отчёты скаутов.

**Влияние:** кто зависит от изменяемых частей (`graphify query "what depends on …"`) и что может сломаться; правки строго точечные — файл целиком не переписывать.

**Решённые развилки:** каждая развилка — решение + одна строка обоснования.

**Шаги:** нумерованный план правок по файлам. Для кодовых задач шаг 1 — тесты, фиксирующие требование/баг (red), дальше код до зелёного.

**Границы:** что НЕ делать (не рефакторить попутно, не трогать генерённое, не выходить за ownership-пути отдела).

**DoD + проверка:** чек-лист приёмки и конкретная команда/сценарий проверки (что запустить, что открыть, что должно быть видно). Задача меняет поведение, структуру или цели → обязательный пункт: обновить затронутые README/доки. Кодовая задача → обязательный пункт: `graphify update .` прогнан (граф актуален).

<!-- Конверт диспатча исполнителю (текст промпта агенту отдела; спеку не дублировать): -->

```
Ты — исполнитель отдела <отдел>. Рабочая директория: <path>.
Твоя спека — тело issue #N: прочитай `gh issue view N` и выполни строго по ней.
За границы спеки не выходи, продуктовых решений не принимай.
По завершении: прогони проверку из DoD, один conventional commit в <ветка/main>
с "closes #N" — НЕ закрывай issue сам, закрытие после приёмки.
Отчёт: изменённые файлы, результат проверки, отклонения от спеки.
```

<!-- Конверт weak-профиля (дешёвая модель-исполнитель) — ДОБАВИТЬ к обычному конверту: -->

```
Профиль: weak. Правила поверх спеки:
— меняй ТОЛЬКО файл из спеки; ожидаемый дифф ≤40 строк, выходит больше — СТОП и доклад;
— контекст в спеке полный (вырезки приложены) — ничего не ищи и не читай сверх него;
— перед отчётом прогони гейты из DoD: <команды lint/typecheck/tests>; verifier'у сдаётся
  только зелёное, вывод гейтов приложи к отчёту;
— чек-лист самопроверки: (1) дифф только в целевом файле, (2) тест шага 1 был красным и
  стал зелёным, (3) комментарии на русском добавлены, (4) graphify update . прогнан;
— никакого рефакторинга и переименований сверх спеки; сомнение = вопрос в отчёте, не импровизация.
```
