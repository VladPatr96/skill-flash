# Промт-пак: ядро

Формат: промт с `{слотами}` → под ним пример заполнения. Копируй, заполняй слоты, отправляй.

## Промты фаундера (управление компанией)

**Поставить цель / стартовать компанию**
```
/office Цель: {что должно получиться, одной фразой результата}
Контекст: {ограничения, стек если важен, куда публикуем, что уже есть}
```
> Цель: русскоязычное медиа о командах Денвера — стабильный контент-конвейер в Telegram
> Контекст: канал уже существует, бот written in Python, репо приватный

**Планёрка**
```
Проведи планёрку: что произошло, какие решения ждут меня, что поедет дальше
```

**Статус**
```
Покажи состояние конвейера одной таблицей: сделано / в работе / blocked чем
```

**Новая задача (issue-first)**
```
Заведи задачу: {что нужно}. Приоритет {high|normal|low}, это {feature|bug|chore}
```
> Заведи задачу: посты про Avalanche выходят без ссылки на трансляцию. Приоритет high, это bug

**Найм / штатное интервью**
```
Проведи штатное интервью: предложи профильные отделы под {функция или направление}
```
> Проведи штатное интервью: предложи профильные отделы под дистрибуцию в VK и Дзен

**Смена приоритетов**
```
Передвинь {#N} в начало очереди, {#M} отложи до {условие}
```

**Месячный обзор**
```
Собери месячный business review: OKR-прогресс, unit-экономика, гейты, решения с «почему»
```

**Переопределить решение оркестратора**
```
По развилке «{тема}» меняю решение на: {решение}. Обнови спеки, которые она задевает
```

## Инженерные промты (по мотивам Anthropic prompt library)

Полная библиотека: https://code.claude.com/docs/en/prompt-library — ~52 слот-промта по фазам discover/design/build/ship/operate. Оркестратор вставляет их в спеки/конверты; самые ходовые в адаптации:

**Сориентироваться в кодовой базе** *(discover/Onboard)*
```
give me an overview of this codebase: architecture, key directories, and how the pieces connect
```

**Оценить влияние до правки** *(discover/Understand — наш кодекс п.2)*
```
what would break if I deleted {target}?
```
> what would break if I deleted the retryWithBackoff helper?

**Объяснить незнакомый код**
```
explain what {path} does and how data flows through it. write it up as {format}
```
> explain what src/scheduler/queue.ts does… write it up as an HTML page with a diagram

**Найти, где происходит поведение**
```
where do we {behavior}?
```
> where do we validate uploaded file types?

**Дебаг по инциденту** *(operate/Debug)*
```
{симптом}: воспроизведи, найди корень, напиши тест, воспроизводящий баг, почини до зелёного
```

**Разбор истории**
```
look through the commit history of {path} and summarize how it evolved and why
```

Правило: промт из библиотеки — сырьё для секции «Шаги»/«DoD» спеки, не замена спеке. Диспатч-конверты исполнителю и верификатору — в `templates/spec-issue.md`.
