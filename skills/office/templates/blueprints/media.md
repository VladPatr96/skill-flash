# Blueprint: Медиа-редакция

**Кому подходит:** новостной/тематический канал или сайт, который регулярно производит контент по инфоповодам (спорт, ниша, город, индустрия).

## Функции бизнеса

поиск инфоповодов → производство контента → редактура → публикация/дистрибуция → аналитика аудитории → engineering конвейера

## Оргпроект (профильные отделы к ядру board/engineering/qa)

| Отдел | Лейбл | Миссия | Агенты (модель) | Скиллы из реестра |
|-------|-------|--------|-----------------|-------------------|
| newsroom | dept:newsroom | находить инфоповоды раньше конкурентов (результаты, травмы/события, слухи, сделки) | news-hunter (Sonnet), скауты-чтецы (Haiku) | deep-research; RSS/API-скрипты отдела (правило трёх) |
| editorial | dept:editorial | тексты по стилю издания: посты, разборы, рубрики | writer (Sonnet), style-editor (Sonnet) | readme-generator-стиль гайдов; STYLE.md издания |
| publishing | dept:publishing | доставка: сетка выпуска, форматирование под платформу, кросспостинг | publisher (Sonnet) | tg-bot-ops; платформенные скрипты |
| analytics | dept:analytics | аудитория и рынок: метрики канала, конкуренты, что заходит | analyst (Sonnet) | pm-metrics, pm-feedback, cc-analytics |
| design (по мере надобности) | dept:design | обложки, визуал рубрик, брендинг | art-worker (Sonnet) | art-director, parallel-design-variants, design-minimal |

Ownership: newsroom/editorial — контентные папки и очереди; publishing — конфиги сетки и интеграции платформ; analytics — company/reports/ метрики; код конвейера — engineering.

## OKR-заготовки

| KR | Метрика | Как меряем |
|----|---------|-----------|
| Регулярность | постов/нед. по сетке без ручных правок | publishing-лог |
| Скорость | медиана «инфоповод → пост» | newsroom-лог |
| Аудитория | подписчики / просмотры / ER | analytics weekly |
| Качество | % постов, прошедших редактуру с 1 раза; ошибок после публикации | editorial/verifier |

## Гейты этапов (роадмап)

0. Стабильный конвейер (сетка без ручного труда N недель) → 1. Рост (дистрибуция, рубрики, кросс-промо) → 2. Сайт/вторая платформа (когда объём контента оправдывает) → 3. Хаб (live, статистика, монетизация).

## Типовые первые задачи

- `type:feature` мониторинг инфоповодов по источникам {список} (newsroom)
- `type:feature` сетка выпуска: {рубрики} × {команды/темы} по расписанию (publishing)
- `type:tooling` еженедельная сводка метрик в отчёт компании (analytics)
- `type:chore` STYLE.md издания: тон, форматы, табу (editorial)

## Промт-пак отдела (слот-формат; копируется в company/<отдел>/prompts.md при найме)

**Мониторинг инфоповодов** *(newsroom)*
```
проверь источники {sources} за {period}: новые инфоповоды по {topics}?
Для каждого: заголовок, суть 1 строкой, источник+ссылка, срочность (молния/день/фон), дубль ли уже опубликованного
```
> sources: ESPN news API, официальные аккаунты клубов, инсайдеры · period: 12 часов · topics: травмы, трейды, результаты

**Пост по инфоповоду** *(editorial)*
```
напиши пост по инфоповоду: {факты+ссылки}. Формат {format} из STYLE.md, длина {len}, язык {lang}.
Факты не выдумывать — только из источников; чего нет в источниках, того нет в посте
```
> format: «молния» · len: до 500 знаков · lang: русский

**Редактура** *(editorial)*
```
отредактируй {draft} по STYLE.md: фактчек по ссылкам источника, тон, структура, длина.
Верни: правки списком + вердикт «в публикацию / на доработку»
```

**Пост в сетку** *(publishing)*
```
подготовь к публикации {post}: форматирование под {platform}, время слота из schedule, медиа {media}.
НЕ публиковать без review-gate, пока канал не делегирован (company/okr.md)
```

**Недельная аналитика** *(analytics)*
```
собери метрики канала за {week}: подписчики Δ, просмотры/пост, ER, топ-3 и антитоп-3 постов.
Выведи: что заходит (форматы/темы/время), 1-3 рекомендации в план следующей недели
```

## Отчёты

Weekly (общий шаблон) + от analytics: блок «метрики канала» в каждый weekly; месячный business review — обязательный с этапа 1.
