# Blueprint: Generic (вывести оргпроект из цели)

**Кому подходит:** бизнес, не попавший в media/saas/creator/agency. Blueprint не даёт готовых отделов — даёт процедуру.

## Процедура оргпроекта

1. Разложи цель по цепочке ценности: **сырьё** (что компания находит/получает) → **производство** (что делает руками) → **контроль качества** → **доставка** (как результат попадает к потребителю) → **обратная связь** (аналитика, деньги, клиенты).
2. Каждое звено с регулярной работой — кандидат в отдел. Разовое — задача в конвейер, не отдел.
3. Для каждого кандидата найди практику (Фаза 4 шаг 2: реестр → awesome-списки → поиск) — чем такие функции решают люди и какие скиллы уже существуют.
4. Собери пакет офферов штатного интервью; спорные отделы предлагай со статусом «отложить — триггер найма: {условие}».

## Ядро всегда

board (CEO) / engineering (если есть код) / qa (verifier). Бизнес без кода: engineering заменяется профильным производственным отделом.

## OKR-заготовки

Минимум три KR: **выпуск** (единицы результата/период), **качество** (% приёмки с первого раза), **эффект** (метрика потребителя: аудитория/выручка/удержание — по природе бизнеса).

## Типовые первые задачи

- `type:chore` карта цепочки ценности + оргпроект (board)
- `type:feature` первый сквозной прогон цепочки вручную — узкие места в issues (все отделы)
- `type:tooling` метрики эффекта: как и кто меряет (analytics-функция)

## Промт-пак

Начни с `templates/prompts-core.md` (промты фаундера + инженерные). Отраслевые промты отдел пишет сам после 2-3 задач: рутина ≥3 раз → слот-промт в company/<отдел>/prompts.md (правило трёх для промтов).
