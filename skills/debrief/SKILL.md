# Debrief

Close the learning loop without leaving open items to drift.

## Procedure

1. Gather facts: closed issues, SHAs, verifier verdicts, blockers, budget use, unit-economics deviations.
2. Give every open item one decision: do now, create issue, defer with trigger/date, or close as irrelevant.
3. Do not move the same item a third time; decide, delete, or escalate.
4. Attribute each delay to one main cause: scope, dependency, quality, environment, or decision.
5. Write transferable lessons to `company/knowledge/lessons.md`; ask before promoting to a global base.
6. Turn process improvements into `type:chore` or `type:tooling` issues.

## Output

Write `company/meetings/YYYY-MM-DD-debrief-<scope>.md` or a weekly report, then update state/OKR if affected.
