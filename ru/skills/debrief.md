# debrief

Используй для weekly/epic retro, закрытия incident или workstream.

Каждый open item получает решение: сделать сейчас, создать issue, отложить с trigger/date или закрыть. Третий перенос запрещён: decide/delete/escalate.

Уроки пишутся в `company/knowledge/lessons.md`; process improvements становятся issues.
