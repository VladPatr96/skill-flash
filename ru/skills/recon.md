# Recon

Цель первых 10 минут — не «понять проект», а построить карту, достаточную для конкретной задачи.

## Шаги

1. **Pulse** — командами, не глазами: `git log --oneline -15` (жив ли; последние изменения — там и жизнь), `git status` (незакоммиченное = чья-то незавершённая работа: оставь на месте и сообщи, никогда не откатывай молча), `ls` (язык, структура, манифесты).
2. **Чтение в строгом порядке**, первый существующий файл на каждой ступени: устав (CLAUDE.md / AGENTS.md / GEMINI.md — локальные правила сильнее твоих привычек) -> README -> манифест (package.json scripts / pyproject / psd1) -> tests/ (по именам видно, что важно) -> CI (.github/workflows — какие проверки обязательны).
3. **Five-line map**, письменно в ответ или журнал: Что делает / Вход / Запуск (точная команда) / Проверка (точная команда) / Опасно (секреты, прод, замороженные зоны). Строка, которую не можешь заполнить, — твоя первая задача; правки начинаются после заполнения Проверки и Опасно.
4. **Task zone** вместо всего кода: 2-3 слова из формулировки задачи -> grep -> 1-2 файла вокруг совпадений -> их импорты. Читай только task zone; чтение проекта «на всякий случай» — сжигание контекста.

## Чеклист выхода

- Команды pulse выполнены, их вывод прочитан.
- Порядок чтения соблюдён без пропусков и перестановок.
- Five-line map записана письменно; Проверка и Опасно заполнены до любой правки.
- Task zone названа: перечислены найденные файлы и их импорты.
