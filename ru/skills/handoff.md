# handoff

Конверт передачи задачи — вручение сессии или подзадачи другому агенту; в отличие от writing-specs (постоянная спека в трекере), конверт — разовый документ передачи.

Конверт самодостаточен: получатель видит только конверт, а не твою сессию — всё нужное кладётся внутрь.

## Шаги

1. Если передан аргумент — это фокус следующей сессии; строй конверт вокруг него.
2. Заполни восемь секций: Context (файлы, ограничения, устав проекта) / Task / Decisions already made / Executor class / DoD + verify command / Report format / Suggested skills / Links to artifacts.
3. Выбери класс исполнителя по типу задачи — huge-context для анализа всего репо, frontier для архитектуры и дебага, cheap для механики, sandbox для запуска кода — и зафиксируй выбор в конверте.
4. На существующие артефакты (PRD, планы, ADR, issues, коммиты) ссылайся путём или URL; конверт указывает на них, а не пересказывает их содержимое.
5. Передавай указатели на места хранения секретов, но никогда не значения: вычищай ключи, пароли и PII до попадания в конверт.
6. Сохрани конверт во временную директорию — рабочее пространство остаётся чистым от разовых документов.
7. Прогони тест самодостаточности — готово, когда свежий агент с нулевым контекстом мог бы начать по одному конверту; закрой каждый найденный пробел и повтори тест.

## Чеклист выхода

- Все восемь секций на месте и заполнены.
- Класс исполнителя зафиксирован с причиной в одну строку.
- В конверте нет значений секретов и дублированного содержимого артефактов.
- Тест самодостаточности пройден.
