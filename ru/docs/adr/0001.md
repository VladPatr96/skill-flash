# ADR-0001: Английский канон, русское зеркало, формат скилла в 50 строк

Статус: принято, 2026-07-07 (решения владельца по умолчанию с планёрки 2026-07-07, эпик #7).

## Контекст

Office начинался как полностью русский репозиторий. Он становится переиспользуемым
публичным плагином для нескольких agent CLI, поэтому нужен один канонический язык,
предсказуемая политика локализации и строгий формат скилла, который дёшево читать,
проверять и переводить. PR эпика #7 должны ссылаться на один постоянный документ,
а не пересказывать эти решения.

## Рассмотренные варианты

- Оставить русский каноном и переводить на английский по запросу — отклонено: блокирует дистрибуцию.
- Вести два равноправных языковых источника — отклонено: гарантированный дрейф, двойная цена ревью.
- Английский канон с русским зеркалом, обновляемым в том же PR — принято.
- Свободный длинный формат скиллов против жёсткого лимита в 50 строк — принят жёсткий лимит.

## Решение

1. Канон — английский: `skills/`, `README.md`, `docs/` и манифесты пишутся на английском.
   Русский — зеркало в `ru/` (`ru/README.md`, `ru/skills/<name>.md`).
2. Синк локализации: EN меняется — русское зеркало обновляется в том же PR;
   исполнитель перевода — модель cheap-класса.
3. Формат скилла: файл целиком <=50 строк. Скелет: frontmatter -> принцип одним
   предложением -> <=7 шагов по 1-2 строки (примеры однострочные, внутри шагов) ->
   exit checklist <=4 проверяемых пунктов. Worked-example секции запрещены.
4. Имена скиллов: 1-2 слова, leading-word тест — имя употребимо глаголом во фразе
   диспатча («прогони done-check»); имя = претрейновый концепт, где возможно.
5. Каждый скилл обязан зафиксировать invocation-решение: model-invoked (`description`
   с EN-триггерами) или user-invoked (`disable-model-invocation: true`).
6. Роутинг моделей в текстах — только классами (frontier / everyday / cheap / huge-context);
   конкретные имена моделей в скиллы не пишутся.
7. Версионирование: завершение эпика #7 = релиз 1.0.0 — смена языка является
   breaking-изменением для русскоязычных пользователей.

## Почему

- Один канонический язык убирает дрейф и делает каждый PR проверяемым по одному источнику.
- Синк зеркала в том же PR превращает локализацию в механический делегируемый шаг, а не бэклог.
- Лимит в 50 строк делает скиллы предсказуемыми для дешёвых исполнителей и быстрыми в проверке.
- Имена-глаголы из претрейновых концептов позволяют диспатчить скилл без пояснений.
- Явное invocation-решение защищает user-only скиллы от случайного срабатывания у модели.
- Роутинг классами переживает смену моделей; имена в тексте устаревают за месяцы.

## Цена, которую платим

- Двойная поддержка EN + RU: каждое user-facing изменение стоит обновления зеркала.
- Миграционный шум для текущих русскоязычных пользователей: меняются пути, имена и язык
  интерфейса — поэтому эпик #7 выходит как breaking-релиз 1.0.0.
