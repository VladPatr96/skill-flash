# Casting

Hiring starts from a concrete use case, not from inventing a role.

## Procedure

1. State the job in one line: who needs to do what, for which issue/spec.
2. Search in order: `skills/office/registry.md`, existing project skills, verified marketplaces/repos, then primary web sources.
3. Run preflight: install command exists, license is compatible, data exposure is acceptable, role does not duplicate an existing owner.
4. Prepare an offer: department, charter, agent/tool, model class, first task, cost vs expected value, risks, fallback.
5. Show a dry run: files, labels, commands, and external access that would be created.
6. Ask approval before installs, new departments, paid services, uploads, or external accounts.
7. Validate with a small safe task before making the hire permanent.

## Output

Return an approved offer, rejected offer, or blocked hiring note with evidence.
